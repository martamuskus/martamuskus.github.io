history

In 1979, Bjarne Stroustrup began developing "C with Classes," inspired by Simula’s features and C’s efficiency. Working at AT&T Bell Labs, he aimed to enhance C for distributed computing, incorporating elements from ALGOL 68, Ada, CLU, and ML. By 1982, "C with Classes" evolved into C++, introducing virtual functions, overloading, references, type-safe memory management, and BCPL-style comments (//). Stroustrup also created the first C++ compiler, Cfront. In 1984, he implemented the first stream I/O library. The first edition of The C++ Programming Language was published in 1985, followed by C++ 2.0 in 1989, adding multiple inheritance, abstract classes, and const member functions. In 1998, C++98 standardized the language, with minor updates in C++03. C++11, released in 2011, brought major improvements, followed by C++14, C++17, and C++20. Stroustrup won the 2018 Draper Prize for his contributions. In 2022, C++ surpassed Java in the TIOBE index, ranking second after Python by 2024

Basic overview

Templates

C++ templates enable generic programming. C++ supports function, class, alias, and variable templates. Templates may be parameterized by types, compile-time constants, and other templates. Templates are implemented by instantiation at compile-time. To instantiate a template, compilers substitute specific arguments for a template's parameters to generate a concrete function or class instance. Some substitutions are not possible; these are eliminated by an overload resolution policy described by the phrase "Substitution failure is not an error" (SFINAE). Templates are a powerful tool that can be used for generic programming, template metaprogramming, and code optimization, but this power implies a cost. Template use may increase object code size, because each template instantiation produces a copy of the template code: one for each set of template arguments, however, this is the same or smaller amount of code that would be generated if the code were written by hand. This is in contrast to run-time generics seen in other languages (e.g., Java) where at compile-time the type is erased and a single template body is preserved.


Object Oriented Programming

C++ extends C with object-oriented programming (OOP) features, including classes that support abstraction, encapsulation, inheritance, and polymorphism. A key distinction is deterministic destructors, enabling the Resource Acquisition is Initialization (RAII) concept.
Encapsulation
Encapsulation hides implementation details, ensuring controlled data access. C++ enforces this via public, protected, and private access specifiers. While C++ supports strict encapsulation through member and friend functions, it also allows more flexible modular programming. Best practices recommend keeping data private or protected while exposing only a minimal public interface.
Inheritance
Inheritance enables a class to acquire properties from another. It can be public, protected, or private, with public inheritance being the most common. Virtual inheritance prevents duplication in multiple inheritance scenarios. Unlike C# or Java, which allow multiple interface inheritance, C++ supports full multiple inheritance. Abstract base classes (ABCs) with pure virtual functions serve a similar role to interfaces in other languages.


Compatibility with C

C++ is often considered to be a superset of C but this is not strictly true. Most C code can easily be made to compile correctly in C++ but there are a few differences that cause some valid C code to be invalid or behave differently in C++. For example, C allows implicit conversion from void* to other pointer types but C++ does not (for type safety reasons). Also, C++ defines many new keywords, such as new and class, which may be used as identifiers (for example, variable names) in a C program. Some incompatibilities have been removed by the 1999 revision of the C standard (C99), which now supports C++ features such as line comments (//) and declarations mixed with code. On the other hand, C99 introduced a number of new features that C++ did not support that were incompatible or redundant in C++, such as variable-length arrays, native complex-number types (however, the std::complex class in the C++ standard library provides similar functionality, although not code-compatible), designated initializers, compound literals, and the restrict keyword. Some of the C99-introduced features were included in the subsequent version of the C++ standard, C++11 (out of those which were not redundant). However, the C++11 standard introduces new incompatibilities, such as disallowing assignment of a string literal to a character pointer, which remains valid C. To intermix C and C++ code, any function declaration or definition that is to be called from/used both in C and C++ must be declared with C linkage by placing it within an extern "C" {/*...*/} block. Such a function may not rely on features depending on name mangling (i.e., function overloading).
